AstroPaper+ поставляется с **оглавлением** (opt-in) для длинных записей. Компонент рендерится дважды из одного источника: сворачиваемый `<details>` в начале статьи на мобильном и липкий `<aside>` в правой колонке (`position: sticky`) на экранах `lg+`. Scroll-spy на базе `IntersectionObserver` подсвечивает текущий заголовок; всё это дружит с View Transitions и плавно деградирует при отключённом JS.

## Включение для отдельной записи

Оглавление подключается opt-in. Поставьте флаг во frontmatter записи:

```md
---
title: "Как добавить липкое оглавление в правой колонке"
pubDatetime: "2026-07-06T00:00:00Z"
description: "..."
tocAside: true # ← добавьте эту строку
---
```

Готово. Флаг добавлен в схему коллекции `posts` в `src/content.config.ts` и по умолчанию равен `false`, так что существующие записи рендерятся ровно как раньше, пока автор явно не включит его.

**Почему opt-in?** Короткие записи выглядят нелепо с боковой панелью из одного пункта. Дополнительно компонент короткое замыкает на «не рендерить», когда в записи меньше 2 заголовков h2/h3 — флаг это подсказка, а не гарантия. Можно увидеть это в действии: в самой этой записи стоит `tocAside: true` и есть 7 h2, поэтому боковая панель отображается; короткая запись вроде `setting-dates-via-git-hooks.md` с 1 h2 не получит оглавления, даже если флаг включён.

## Как это рендерится

Ниже `lg` (1024px) оглавление — это сворачиваемый `<details>` в начале статьи, над заголовком. Нажмите на summary, чтобы раскрыть список заголовков. Элемент `<details>` доступен по умолчанию — никакого JS не нужно, переключается с клавиатуры, дружелюбен к скринридерам.

На `lg+` те же данные рендерятся как липкий `<aside>` в правой колонке:

- Ширина: `w-56` (224px) — достаточно для двух строк текста заголовка на большинстве шрифтов
- Позиция: `position: sticky; top: 5rem;` — панель остаётся приклеенной к верху вьюпорта при прокрутке
- Максимальная высота: `calc(100vh - 6rem)` с внутренним `overflow-y-auto`, чтобы очень длинные списки заголовков прокручивались внутри панели, а не вылезали за вьюпорт
- Ширина: `w-56` (224px) — достаточно для двух строк текста заголовка на большинстве шрифтов

Когда флаг включён, страница записи также подстраивает окружающую вёрстку: `<main>` расширяется с `max-w-3xl` (768px) до `max-w-[68rem]` (1088px) на `lg+`, а статья + панель сидят в 2-колоночной CSS-сетке (`grid-cols-[1fr_minmax(0,_1fr)]`). `min-w-0` статьи не даёт длинным блокам кода взорвать сетку; панель занимает свои `w-56`, остальное уходит статье. Ниже `lg` сетка схлопывается в одну колонку, панель скрывается, и в дело вступает мобильный `<details>`.

Активный заголовок в панели подсвечивается акцентным цветом, жирным начертанием и левой границей (`border-s-2 border-accent`). Компонент также выставляет `aria-current="location"` на активной ссылке для ассистивных технологий.

## Scrollspy

Активное состояние управляется `IntersectionObserver`, который следит за каждым h2 и h3 внутри статьи. `rootMargin` — `0px 0px -75% 0px`, то есть заголовок считается «активным», пока он находится в верхней четверти вьюпорта — совпадает с интуицией читателя: вы только что проскроллили его или читаете прямо сейчас.

Observer пересоздаётся на каждом `astro:page-load` и сносится на `astro:before-swap`, так что scrollspy переживает view transitions Astro без утечки observer'ов между переходами страниц. Observer — пассивный потребитель: он никогда не вызывает `preventDefault` на прокрутку, не двигает страницу, не конфликтует с браузерным восстановлением скролла. Клик по ссылке в оглавлении просто переходит на существующий `#anchor` URL; нативный якорный скролл браузера делает остальное.

Если вы предпочитаете обойтись без подсветки, управляемой движением, медиа-запрос `prefers-reduced-motion` в CSS компонента убирает переход. Сама подсветка — переключение класса, а не анимация, поэтому она всё равно появляется мгновенно.

## Настройка глубины заголовков

Компонент фильтрует входные данные до h2 и h3. Заголовки h4+ намеренно исключены — они делают панель шумной в глубоко вложенных записях. Если вы хотите разрешить h4, отредактируйте фильтр в `src/components/TableOfContents.astro`. Сейчас он:

```ts
const minLevel = 2; // h2
const maxLevel = 3; // h3
```

Поменяйте `maxLevel` на `4`, чтобы поднять порог.

## i18n метки

Метка боковой панели берётся из `t.post.onThisPage` в каждой локали:

| locale | label        |
| ------ | ------------ |
| en     | On this page |
| ru     | Содержание   |
| tr     | Bu sayfada   |

Метка используется как заголовок секции и как `<nav aria-label>`, так что скринридеры объявляют её корректно. Чтобы добавить новый язык, обновите `src/i18n/types.ts`, соответствующий файл `src/i18n/lang/<code>.ts` и `LOCALES` в `src/i18n/locales.ts` — тот же процесс, что и для остальной части шаблона.

## Как это взаимодействует с существующим инлайн-списком `remark-toc`

AstroPaper+ также запускает `remark-toc` в общем конвейере Markdown, что позволяет авторам вставить заголовок `## Table of contents` где угодно в записи, чтобы получить автогенерируемый инлайн-список. Эти две возможности независимы:

- `remark-toc` (инлайн-список) всегда включён. Он срабатывает только в записях, где есть заголовок `## Table of contents`.
- `tocAside` (боковая панель) подключается для каждой записи отдельно. Он срабатывает в записях, где стоит флаг.

Можно использовать оба одновременно (инлайн-список показывается под заголовком `## Table of contents`; боковая панель плавает справа), но обычно это избыточно. Паттерн, который использует эта запись — `tocAside: true` и без заголовка `## Table of contents` — самый чистый для длинных записей. Для более коротких записей обычно достаточно инлайн-списка, и флаг можно не ставить.
