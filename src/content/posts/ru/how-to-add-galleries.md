AstroPaper+ поставляется с необязательной коллекцией контента **galleries** (см. issue #553). Каждая галерея — это отдельный MDX-файл со списком изображений; страница детали рендерит адаптивную сетку миниатюр, при клике открывающую каждое изображение в лайтбоксе **PhotoSwipe v5** с навигацией с клавиатуры, зумом по свайпу/щипку и предзагрузкой соседних кадров.

## Включение функции

Функция **opt-in** — по умолчанию она отключена, чтобы существующие сайты не получали неожиданный новый маршрут верхнего уровня. Переключите тумблер в `astro-paper.config.ts`:

```ts
features: {
  // ... существующие флаги ...
  enableGalleries: true,
},

```

Тумблер управляет сразу тремя вещами:

- создаются страницы списка `/galleries/` и детали `/galleries/<slug>`,
- в навигации шапки появляется пункт «Галереи»,
- переключатель языка протаскивает URL галерей через текущую страницу во все поддерживаемые локали.

Когда тумблер в `false`, маршруты возвращают пустой `getStaticPaths()`, так что никаких «осиротевших» URL не попадает в sitemap и ниоткуда на них не ссылаются.

## Структура каталогов

```
src/content/galleries/
  en/
    sample-walk.mdx
    japan-2025.mdx
  ru/
    sample-walk.mdx
    japan-2025.mdx
  tr/
    ...
  _assets/         # ← общие изображения, игнорируются загрузчиком
    cover.svg
    photo-1.jpg
    photo-2.jpg

```

Локаль определяется по ведущей папке — то же соглашение, что и для `posts/` и `projects/`. Папки, начинающиеся с `_`, пропускаются glob-загрузчиком, поэтому `_assets/` — безопасное общее хранилище.

## Создание галереи

Frontmatter соответствует схеме коллекции `galleries`:

```mdx
---
title: "Япония 2025"
description: "Три недели в Токио, Киото и Осаке."
pubDatetime: "2026-04-12T08:30:00.000Z"
featured: true
coverImage: ../_assets/japan-cover.jpg
images:
  - src: ../_assets/japan-01.jpg
    alt: "Токийский горизонт со смотровой Shibuya Sky"
    caption: "День 1 — Сибуя в сумерках"
  - src: ../_assets/japan-02.jpg
    alt: "Тории храма Фусими Инари"
    caption: "День 4 — Фусими Инари на рассвете"
  - src: ../_assets/japan-03.jpg
    alt: "Сакура вдоль философской тропы"
    caption: "День 9 — цветущий Киото"
---

Тело файла — обычный MDX: текст-история, подписи, ссылки, даже
другие встраивания. Сетка миниатюр рендерится автоматически по
списку `images` под вашей прозой.
```

Примечания:

- `images` **обязательно** (схема требует `z.array(...).min(1)`).
- `alt` у каждого изображения **обязателен** — PhotoSwipe использует его как метку доступности и как запасную подпись.
- `width` / `height` для каждого изображения — необязательные подсказки. Хелпер `image()` схемы Astro извлекает реальные размеры из файла, так что их можно опускать.
- `coverImage` — то, что показывается на **карточке** галереи (в списке `/galleries/`). Если его пропустить, на карточке будет заглушка «No cover».
- `featured: true` поднимает галерею наверх списка — тот же флаг, что и в `posts/` и `projects/`.
- `draft: true` скрывает галерею в продакшене (в `dev` она видна всегда — то же правило, что и для записей).
- `pubDatetime` + `timezone` учитывают ту же настройку `content.scheduledPostMargin`, что и для записей — материалы, дата которых дальше этой границы в будущем, удерживаются.

## Как работает лайтбокс

PhotoSwipe v5 динамически импортируется по требованию через `src/scripts/galleryLightbox.ts`. CSS грузится только на странице детали галереи (включается в стили страницы через side-effect-импорт). Ни то, ни другое не попадает на остальной сайт — главная, страницы записей и раздел проектов не платят ни байта за UI галереи.

Сетка миниатюр использует разметку `data-pswp-*`, так что пользователи без JavaScript всё равно получают рабочее открытие полного изображения в новой вкладке (`target="_blank" rel="noopener"`). Когда JS включён, PhotoSwipe перехватывает клик и открывает на странице оверлей с:

- навигацией стрелками клавиатуры / свайпом между изображениями,
- зумом щипком / двойным тапом,
- закрытием кнопкой или кликом вне,
- предзагрузкой следующего и предыдущего изображения.

Лайтбокс переинициализируется после каждой view transition Astro (обработчик `astro:after-swap`), так что клиентская навиляция между галереями сохраняет интерактивность без полной перезагрузки.

## i18n

Все UI-строки переведены и проходят через уже используемую систему плейсхолдеров `tplStr`:

| ключ                                                        | назначение                                  |
| ----------------------------------------------------------- | ------------------------------------------- |
| `nav.galleries`                                             | метка в навигации шапки                     |
| `pages.galleriesTitle` / `galleriesDesc` / `galleriesEmpty` | h1 списка, вступление, состояние «пусто»    |
| `gallery.viewGallery`                                       | CTA на карточке                             |
| `gallery.openGallery`                                       | aria-label на ссылке карточки               |
| `gallery.backToList`                                        | ссылка «назад» на странице                  |
| `gallery.ofLabel`                                           | счётчик «N из M» внутри лайтбокса           |
| `gallery.photoCount` (`PluralForms`)                        | счётчик на карточке (CLDR через `plural()`) |

Чтобы добавить новый язык, нужно обновить `src/i18n/types.ts`, соответствующий файл `src/i18n/lang/<code>.ts` и `LOCALES` в `src/i18n/locales.ts` — тот же процесс, что и для остальной части шаблона.
