Обновлять зависимости проекта — занятие нудное. Однако пренебрегать обновлениями тоже не стоит 😬. В этом посте Sat Naing делится тем, как он обычно обновляет свои проекты, на примере AstroPaper+ (форк AstroPaper+). Эти же шаги подходят и для других JS/Node-проектов.

## Обновление зависимостей пакета

Есть несколько способов обновить зависимости; Sat Naing перепробовал разные методы, чтобы найти самый простой. Один из них — вручную обновлять каждый пакет через `npm install package-name@latest`. Этот метод самый прямолинейный. Однако он не всегда самый эффективный.

Рекомендуемый Sat Naing способ — использовать пакет [npm-check-updates](https://www.npmjs.com/package/npm-check-updates). По нему есть хорошая [статья](https://www.freecodecamp.org/news/how-to-update-npm-dependencies/) на freeCodeCamp, так что подробности о том, что это и как использовать, здесь не раскрываются. Вместо этого в посте разбирается типичный подход, который Sat Naing использует.

Сначала установите `npm-check-updates` глобально:

```bash
npm install -g npm-check-updates
```

Перед любыми обновлениями полезно проверить, какие новые версии доступны:

```bash
ncu
```

Чаще всего патч-версии можно обновлять без какого-либо влияния на проект. Sat Naing обычно обновляет патчи одной из команд: `ncu -i --target patch` или `ncu -u --target patch`. Разница в том, что `ncu -u --target patch` обновит все патчи сразу, а `ncu -i --target patch` даст выбор, какие пакеты обновлять. Какой подход выбрать — решайте сами.

Следующая часть — обновление минорных версий. Минорные обновления пакетов обычно не ломают проект, но всегда полезно заглянуть в release notes соответствующих пакетов. В таких релизах часто появляются интересные возможности, применимые в проекте:

```bash
ncu -i --target minor
```

Наконец, в зависимостях могут быть и мажорные обновления. Проверьте оставшиеся обновления командой:

```bash
ncu -i
```

Если есть какие-то мажорные обновления (или другие, которые ещё предстоит сделать), команда выше выведет эти оставшиеся пакеты. Если обновление мажорное, нужно быть очень осторожным — скорее всего, оно сломает весь проект. Поэтому внимательно прочитайте соответствующие release notes (или документацию) и внесите нужные правки. Если после `ncu -i` больше обновлять нечего — _**Поздравляю!!!**_ — все зависимости в проекте успешно обновлены.

## Обновление шаблона AstroPaper+

Как и другие open-source-проекты, AstroPaper+ развивается: исправляются баги, добавляются функции и т.д. Если вы используете AstroPaper+ как шаблон, вы можете подтянуть свежие изменения либо из этого форка (`msoltanov/astro-paper-plus`), либо из upstream AstroPaper+ (`satnaing/astro-paper`) с помощью того же git-процесса. Дело в том, что вы наверняка уже кастомизировали шаблон под себя. Поэтому Sat Naing не может показать «один идеальный способ на все случаи», чтобы обновить шаблон до последнего релиза. Но вот несколько советов, как обновить шаблон, не сломав свой репозиторий. Имейте в виду, что чаще всего обновления зависимостей пакетов бывает достаточно.

### Файлы и каталоги, о которых стоит помнить

В большинстве случаев файлы и каталоги, которые вы, скорее всего, не хотите перетирать (потому что уже их изменяли): `src/content/posts/<locale>/`, `astro-paper.config.ts`, `src/content/pages/<locale>/about.md`, а также другие ассеты и стили вроде `public/` и `src/styles/`. Если вы обновляете только минимум шаблона, можно заменить всё на последнюю версию AstroPaper+, кроме указанных файлов и каталогов. Это похоже на чистый Android и проприетарные оболочки вроде OneUI. Чем меньше вы меняли базу, тем меньше потом придётся обновлять.

Можно заменить каждый файл вручную, а можно использовать git, чтобы обновить всё разом. Ручная замена файлов по одному в этом посте не рассматривается — лучше взять от git всё, что можно.

### Обновление шаблона AstroPaper+ с помощью git

Сначала добавим upstream-репозиторий в качестве удалённого. Можно использовать этот форк (`msoltanov/astro-paper-plus`), а можно ссылаться на оригинальный upstream:

```bash
# Upstream AstroPaper+
git remote add astro-paper <upstream-url>
# Или этот форк
git remote add astro-paper-plus https://github.com/msoltanov/astro-paper-plus.git
```

Переключитесь на новую ветку, чтобы обновить шаблон. Если вы знаете, что делаете, и уверены в своём владении git, этот шаг можно пропустить:

```bash
git checkout -b build/update-astro-paper-plus
```

Затем подтяните изменения из удалённого репозитория:

```bash
git pull astro-paper-plus main

# или

git pull astro-paper main
```

Если получите ошибку `fatal: refusing to merge unrelated histories`, разрешите её так:

```bash
git pull astro-paper-plus main --allow-unrelated-histories
```

После выполнения этой команды вы, скорее всего, столкнётесь с конфликтами в проекте. Их нужно разрешить вручную и внести нужные правки под свои нужды. После разрешения конфликтов тщательно протестируйте блог, чтобы убедиться, что всё работает как ожидается. Проверьте статьи, компоненты и любые кастомизации, которые делали. Когда будете довольны результатом, можно влить ветку обновления в main (если обновляете шаблон в отдельной ветке). Поздравляю! Шаблон успешно обновлён до последней версии. Ваш блог теперь в актуальном состоянии и готов сиять! 🎉
